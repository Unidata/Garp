What's New in Version 2.02


 

GARP Version 2.02 is a minor bug release with limited added functionality. Bug fixes are listed below.
 

Bug Fixes

  • Some model levels and layers did not display in the level popup dialog. Now they do.
  • Some widget attachments were not properly alligned when using the CDE window manager. Now, they are.
  • Fixed a bug that caused GARP to crash if you loaded a topographic image and then immediatly cleared the screen without loading any other depictables.
  • Fixed a bug that caused GARP to crash if you load a plan view depictable and then immediately load (without clearing) a cross-section vector depictable.
  • Fixed a bug that caused the titles of all the model vertical projections to be the wrong color when a title was toggled off.
  • Redefined how default directories are built in dialogs.c to define a default chapter for each projection. These previously existed but were broken at some point. Did not add any keys to Garp_defaults but this change affects the keys: fdf_directory, xsfdf_directory, thfdf_directory, and vpfdf_directory. Previously, the corresponding root directory was explicitly part of the these directory definitions in Garp_defaults. Now, they are implicit, meaning, they are used to define the root directory even though they are not included in the default key definitions.
  • A few memory leaks plugged.
