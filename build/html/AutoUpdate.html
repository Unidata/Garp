GARP Auto-Update

Auto-Update

When auto-update is turned on, GARP will look at what you have loaded and reload it if new data arrives. New data are added at the expense of the oldest data in the loop. That is, if two new frames are added, the oldest two frames are removed. Thus, the number of frames in the loop will be constant. Existing data will also be refreshed (redrawn) if new data has arrived since the last update cycle. For example, surface data will be replotted if new station reports have arrived since the last update.

The button at the top of this dialog allows you to toggle auto-update on or off. With the slider widget below this button you can adjust the update frequency (in minutes). The default setting for auto-update and the update interval are configurable in "Garp_defaults" using the keys autoupdate and update_timer.

Note that if you have more than one data object loaded (EG; surface data over a satellite image), then time matching must be ON to have all the objects updated.
